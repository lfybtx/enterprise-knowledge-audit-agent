# 界面功能说明

本文档根据当前工作台界面截图整理，用于说明演示时可以直接观察到的产品能力。当前默认登录账号为 `admin / admin123456`；旧版 Alice/Bob 演示账号已经移除。

## 审计工作台

首页是“企业知识库审计 Agent”工作台，主要区域包括：

- 顶部登录状态、退出入口和服务健康状态；
- “审计工作台”“注册”“知识库管理”三个主要页面入口；
- 审计问题输入框和常用演示问题；
- 当前用户、当前角色、已索引文档数和审计记录数；
- 运行审计和加载冲突案例两个操作入口。

系统会基于当前用户可见的知识库证据回答问题，避免跨用户或跨知识库直接使用不可见文档。

## 文档接入

文档接入区支持选择知识库、填写文档标题并上传文件。截图中的示例展示了：

- 将文档写入指定知识库；
- 支持 Word 等制度文件导入；
- 上传过程中显示“上传中”、处理状态和重试次数；
- 上传失败或任务重试时向用户反馈处理进度。

同一区域还支持填写网页标题和 URL，点击“抓取网页”将 HTML 页面接入知识库。

## 审计结果

审计结果页由以下部分组成：

- 评测基线卡片：题目数、Recall@1、引用准确率和平均延迟；
- 带证据回答：回答中的结论绑定 `[证据1]` 等引用；
- 风险发现：按 High、Medium 等等级展示风险、依据和建议动作；
- 报告导出：支持 Markdown 和 PDF；
- 人工复核：支持“确认并回写修正”和“驳回”。

截图中的示例识别出“未授权导出完整客户名单”的 High 风险，并同时发现导出文件保存和传输风险、未授权导出后的应急响应风险。该结果说明系统不仅返回答案，也会将制度冲突转换为可处理的风险清单。

## 推荐演示顺序

1. 登录并展示当前用户、角色、服务状态和知识库数量。
2. 输入客户数据导出问题，运行审计。
3. 展示带引用回答和 High/Medium 风险发现。
4. 导出 Markdown 或 PDF 审计报告。
5. 使用“确认并回写修正”或“驳回”演示人工复核闭环。
6. 返回审计历史，查看已保存的结果和 Workflow Trace。
7. 上传制度文件或抓取网页，展示知识库接入和异步处理状态。
