# 第十四课：审计历史回放

本课把 trace 从“当前页面结果”扩展成“可回放历史”。

## 页面现在有两种 trace

1. 当前 trace
2. 历史 trace

当前 trace 来自刚刚执行的 `/api/ask` 或 `/api/reports/export`。
历史 trace 来自 `/api/audit-log`，可以从列表里点开。

## 为什么要这样设计

面试演示时，单次请求的 trace 只能说明“这次是怎么跑的”。
历史列表能说明：

- 这个用户最近都做了什么
- 哪些问题触发了审计
- 哪些 trace 的耗时更长
- 哪些导出和问答属于同一条链路

## 当前页面行为

- 右侧先展示最近审计记录列表
- 点击某条记录后，下面的 trace 面板会切换到该条记录
- 当前问答完成后，trace 面板会自动显示这次请求的 trace

## 后端数据来源

- 没有数据库时，`audit_log` 仍然保存在内存里，方便本地演示
- 有数据库时，`workflow_runs` 和 `workflow_trace_steps` 会提供历史回放
