# 第十一课：前端用户切换

本课完成多用户隔离的演示入口。页面顶部新增用户选择器，切换用户后，前端会把用户身份放到请求头里。

## 用户选择器

当前提供三个演示用户：

```text
local-demo
demo-alice
demo-bob
```

选择结果会保存到浏览器 `localStorage`，刷新页面后仍会保留上次选择。

## 请求头如何传递

前端统一通过 `authHeaders()` 生成请求头：

```js
{
  "X-User-Id": "demo-alice"
}
```

以下请求都会带上当前用户：

- 文档列表：`GET /api/documents`
- 上传文档：`POST /api/documents/upload`
- 审计问答：`POST /api/ask`
- 导出报告：`POST /api/reports/export`
- 评测接口：`POST /api/evaluate`

## 切换用户后为什么清空结果

审计结果是基于某个用户可见的证据生成的。切换用户后，如果继续展示上一个用户的回答，就容易误解为当前用户也有权限看到这些证据。

因此页面在切换用户时会：

1. 保存新用户。
2. 清空当前审计结果。
3. 重新加载当前用户可见文档。

## 演示方式

1. 选择 Alice，上传一份文档。
2. Alice 的文档列表里可以看到该文档。
3. 切换 Bob，文档列表里看不到 Alice 的文档。
4. Bob 用同样的问题查询时，后端不会检索 Alice 的知识库。

这就是多用户权限隔离的最小可演示闭环。
