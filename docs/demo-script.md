# 面试演示脚本

这份脚本用于面试或项目展示。目标是让面试官在 10-15 分钟内看懂：这个项目不是普通 RAG 问答，而是一个具备审计、证据、权限、评测和运维能力的 Agent 应用。

## 1. 启动项目

```powershell
copy .env.example .env
docker compose up -d --build
```

打开：

```text
http://127.0.0.1:8000
```

可以先说明：

- 后端是 FastAPI。
- 数据库是 PostgreSQL + pgvector。
- 原始文件进入 MinIO。
- Embedding 和 reranker 默认用本地开源模型，不依赖 API Key。
- Chat LLM 可以接 DeepSeek，但不可用时会 fallback。

## 2. 登录 Alice

使用页面顶部登录框：

```text
用户名：alice
密码：alice123456
```

说明 Alice 是 editor，能上传文档、提问、导出报告、审批自己的高风险审计记录。

## 3. 展示知识库内容

页面下方有“当前知识库可见文档”。这里可以说明：

- 文档上传后会解析、切片、生成 embedding。
- 文档和 chunk 存在 PostgreSQL。
- 原始文件存储在 MinIO。
- 每个 chunk 带来源位置，后续引用不会只给文件名。

## 4. 运行一次审计问题

推荐问题：

```text
销售是否可以直接导出完整客户名单？请说明风险和正确流程。
```

点击“运行审计”。

重点展示：

- 回答不是自由发挥，而是来自检索证据。
- 每条证据有标题、来源、位置、摘录和排序分数。
- 风险发现会给风险等级、依据和建议动作。

## 5. 展示冲突识别

推荐问题：

```text
旧版销售工具是否可以直接下载完整客户名单？请说明它和当前制度的冲突，并给出整改建议。
```

重点说明：

- 普通 RAG 往往只回答“可以”或“不可以”。
- 这个系统会发现旧规范和当前制度存在冲突。
- 冲突类和高风险类结果会进入人工确认节点。

## 6. 查看工作流 Trace

在结果区域查看“工作流追踪”。

说明每一步都会记录：

- 检索 Agent：关键词、向量、融合、reranker。
- 审计 Agent：规则识别风险和冲突。
- 报告 Agent：生成回答和风险报告。
- 人工确认节点：判断是否需要审批。

Trace 里还会记录：

- prompt
- tool calls
- duration
- token 估算
- failure reason
- LLM 调用状态

## 7. 演示人工审批

如果问题触发高风险或冲突，页面会出现审批区域。

可以点击：

- 批准报告
- 驳回报告

然后在审计历史中查看审批状态变化。

## 8. 演示权限隔离

切换到 Bob 或使用 Bob 登录：

```text
用户名：bob
密码：bob123456
```

说明：

- Bob 是 viewer。
- Bob 不能上传文档。
- Bob 不能访问系统诊断。
- 文档列表和审计历史会按当前用户过滤。

## 9. 上传样例文档

可以上传 `data/sample_uploads` 下的文件：

- `客户数据导出管理规范.pdf`
- `客户数据导出补充规范.docx`
- `客户数据导出审批清单.xlsx`

上传后说明：

- PDF 会提取页码。
- Word 会提取段落。
- Excel 会提取 Sheet 和 Row。
- 扫描 PDF 可以走 OCR。

## 10. 导出报告

在结果页点击：

- Markdown
- PDF

说明：

- 报告不是截图，而是结构化风险报告。
- PDF 已处理中文字体，避免乱码。

## 11. 展示系统状态

打开“系统状态与索引诊断”。

重点说明：

- 数据库是否连接。
- pgvector 是否启用。
- 文档数、切片数、审计记录数、Trace 步骤数。
- 是否存在缺失 embedding 的切片。
- 是否存在重复文档。
- 当前 embedding 和 LLM 配置。

这部分体现工程可运维性。

## 12. 展示评测结果

页面有评测面板，也可以打开：

```text
docs/evaluation-report.md
```

说明当前有 60 条评测用例，覆盖：

- 检索命中率。
- 引用准确率。
- 回答质量。
- 风险识别。
- 冲突识别。
- 证据绑定。
- 审批触发。

## 13. 面试总结话术

可以这样总结：

```text
这个项目覆盖了企业 Agent 岗位常见 JD 里的 RAG、向量数据库、多 Agent 编排、权限隔离、评测、可观测、Docker 部署和报告导出。相比普通文档问答，它强调证据、风险、冲突和审计流程，比较贴近企业真实落地。
```
