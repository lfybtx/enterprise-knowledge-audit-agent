# 项目状态与后续路线图

更新时间：2026-07-14

## 当前状态

项目当前已具备可用于演示和内测的企业知识库审计 Agent 能力，包括：

- 文档解析、切片、引用定位和多格式导入；
- 关键词检索、向量检索、融合排序和 reranker；
- 风险识别、制度冲突识别、结构化报告和人工复核；
- JWT 登录、知识库成员角色和文档可见范围控制；
- 审计历史、Workflow Trace、报告导出和系统诊断；
- Docker Compose 部署、PostgreSQL/pgvector、MinIO 和 Redis/RQ 异步任务；
- 60 条评测用例及可复现评测结果。

## 最近验证结果

本地虚拟环境验证结果：

```text
80 passed, 8 skipped
```

前端 JavaScript 语法检查：通过。

跳过的测试主要包括：

- 6 个 PostgreSQL/SQLAlchemy 相关测试，因为当前本地虚拟环境未安装 SQLAlchemy；
- 2 个权限测试，需要启动 PostgreSQL 集成环境后执行。

本次未完成 Docker Compose 集成验证，原因是当前环境无法访问 Docker Engine。部署前仍需实际验证数据库迁移、MinIO、Redis/RQ、worker 和健康检查链路。

## 上线前待办

### P0：安全与运行可靠性

1. 生产环境强制配置 `JWT_SECRET`，禁止使用默认密钥。
2. 为数据库、MinIO 和其他服务增加生产配置校验，禁止使用默认密码启动。
3. 为文件上传增加大小限制、扩展名白名单、内容类型校验和异常文件处理。
4. 为 URL 导入增加 SSRF 防护，禁止访问本机、内网、云元数据地址，并校验重定向目标。
5. 启动完整 Docker 环境，执行 PostgreSQL 权限隔离、数据持久化、MinIO 存储和异步任务测试。

### P1：交付质量

1. 完善异步上传的任务状态轮询、失败重试、幂等控制和临时文件清理。
2. 增加结构化日志、请求 ID、任务失败告警和模型调用指标。
3. 统一 API 错误响应格式和错误码，并补充 OpenAPI 请求/响应示例。
4. 补充前端上传进度、任务失败提示、网络异常处理和重试入口。
5. 将集成测试纳入 CI，确保数据库和权限测试不会长期处于跳过状态。

### P2：能力增强

1. 扩展评测集，覆盖权限绕过、长文档、扫描 PDF、恶意 URL 和模型失败场景。
2. 为 Recall、引用准确率、风险识别准确率设置 CI 回归阈值。
3. 增加 OCR 质量诊断、表格还原和低质量文本告警。
4. 对 TopK、融合权重和 reranker 参数进行离线调优。
5. 将演示级权限模型逐步扩展为完整租户、部门和文档 ACL 管理。

## 推荐实施顺序

```text
生产安全配置
    -> Docker 集成验证
    -> 上传与 URL 安全
    -> 异步任务闭环
    -> 监控与 API 规范
    -> 前端体验完善
    -> 检索、OCR 与评测优化
```
